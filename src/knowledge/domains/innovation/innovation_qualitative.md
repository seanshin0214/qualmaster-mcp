## 혁신 연구와 질적연구

### 왜 질적연구가 중요한가?

```
혁신의 특성:
├── 불확실성과 모호성
├── 창발적 과정
├── 맥락 의존성
├── 복잡한 상호작용
├── 시간에 따른 진화
└── 희소 사건 (파괴적 혁신)

질적연구 강점:
├── 혁신 과정 추적
├── 맥락 속 이해
├── 새로운 현상 탐구
├── 인과 메커니즘 규명
└── 실패 분석
```

### 주요 연구 영역

```
1. 혁신 과정 (Innovation Process)
├── 아이디어 생성
├── 개발 과정
├── 상업화
└── 확산

2. 조직 혁신 (Organizational Innovation)
├── 혁신 문화
├── 양손잡이 조직
├── 개방형 혁신
└── 혁신 저항

3. 파괴적 혁신 (Disruptive Innovation)
├── 파괴 메커니즘
├── 기존 기업 대응
├── 신생 기업 전략
└── 산업 변화

4. 사용자 혁신 (User Innovation)
├── 리드 유저
├── 공동 창조
├── 크라우드소싱
└── 오픈 소스

5. 사회 혁신 (Social Innovation)
├── 사회적 가치
├── 시스템 변화
├── 확산 장벽
└── 제도 변화
```

## 주요 이론적 프레임워크

### 1. 파괴적 혁신 이론 (Christensen)

```
핵심 개념:
├── 존속적 vs 파괴적 혁신
├── 가치 네트워크
├── 자원-프로세스-가치
└── 혁신자 딜레마

질적연구 질문:
├── 파괴는 어떻게 시작되는가?
├── 기존 기업은 왜 대응에 실패하는가?
├── 경계 조건은 무엇인가?
└── 산업마다 다른 패턴은?
```

### 2. 양손잡이 조직 (Ambidexterity)

```
핵심 개념:
├── 탐험 (Exploration)
├── 활용 (Exploitation)
├── 구조적 vs 맥락적 양손잡이
└── 균형과 긴장

질적연구 접근:
├── 균형 달성 과정
├── 긴장 관리 방식
├── 조직 설계 진화
└── 리더십 역할
```

### 3. 개방형 혁신 (Chesbrough)

```
핵심 개념:
├── 내부 → 외부 (Inside-out)
├── 외부 → 내부 (Outside-in)
├── 비즈니스 모델 혁신
└── 생태계 관점

질적연구 주제:
├── 개방 과정과 저항
├── 파트너십 역동
├── 지식 흐름
└── 문화 변화
```

### 4. 혁신 확산 (Rogers)

```
핵심 개념:
├── 채택 곡선 (S-curve)
├── 채택자 유형
├── 혁신 특성
└── 커뮤니케이션 채널

질적연구 확장:
├── 채택 결정 과정
├── 맥락적 요인
├── 재발명 (Re-invention)
└── 저항과 거부
```

## 권장 방법론

### 1. 종단적 사례연구 (Longitudinal Case)

```
적합한 상황:
├── 혁신 과정 추적
├── 시간에 따른 변화
├── 인과 메커니즘
└── 경로 의존성

설계:
├── 실시간 추적 + 회고적 재구성
├── 다중 데이터 소스
├── 여러 시점 데이터
└── 과정 분석
```

### 2. 비교 사례연구 (Comparative Case)

```
적합한 상황:
├── 성공 vs 실패
├── 혁신 유형 비교
├── 산업 간 비교
├── 맥락 효과

분석:
├── 복제 논리
├── 패턴 매칭
├── 경쟁 설명 검증
└── 교차 사례 분석
```

### 3. 과정 연구 (Process Research)

```
핵심 질문:
├── 어떻게 (How)
├── 왜 (Why)
├── 시간에 따른 전개
└── 메커니즘

분석 기법:
├── 사건 순서 분석
├── 시간적 브래킷팅
├── 시각적 맵핑
└── 과정 모델 개발
```

### 4. 역사적 분석 (Historical Analysis)

```
적합한 상황:
├── 산업 변화
├── 기술 궤적
├── 장기 혁신 패턴
└── 제도적 진화

방법:
├── 아카이브 연구
├── 구술 역사
├── 2차 자료 분석
└── 시계열 재구성
```

## 글쓰기 전략

### Research Policy 스타일

```
특징:
├── 정책 함의 중시
├── 이론적 기여
├── 체계적 분석
└── 장기 관점

구조:
1. Introduction: 정책적 중요성
2. Literature: 이론적 기반
3. Method: 투명한 절차
4. Case: 풍부한 기술
5. Analysis: 체계적 분석
6. Discussion: 정책 함의
```

### 혁신 사례 기술

```
좋은 사례 기술:
├── 시간 순서 명확
├── 핵심 결정 강조
├── 맥락 정보 충분
├── 다양한 관점 포함
└── 분석과 연결

피해야 할 것:
├── 단순 연대기
├── 성공 스토리만
├── 정치적 서술
├── 분석 없는 묘사
└── 과도한 세부사항
```
