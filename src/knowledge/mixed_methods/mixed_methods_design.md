# Mixed Methods 연구 설계
## 혼합연구방법론 완전 가이드

---

## Mixed Methods란?

### 정의 (Creswell & Plano Clark, 2018)

```
Mixed Methods Research:
"연구자가 질적, 양적 방법, 접근, 개념을
 단일 연구 또는 연속된 연구에서 결합하는 것"

핵심 특징:
├── 질적 + 양적 데이터 수집
├── 두 데이터 유형의 분석
├── 의도적 통합
├── 결합에서 나오는 추론
└── 연구 질문에 기반한 설계
```

### 왜 Mixed Methods인가?

```
장점:
├── 보완성 (Complementarity)
│   └── 각 방법의 약점 보완
├── 완전성 (Completeness)
│   └── 더 완전한 이해
├── 개발 (Development)
│   └── 한 방법이 다른 방법 정보 제공
├── 확장 (Expansion)
│   └── 연구 범위 확장
└── 확증 (Corroboration)
    └── 삼각화를 통한 검증

단점/도전:
├── 두 방법론 숙달 필요
├── 더 많은 시간/자원
├── 통합의 어려움
├── 패러다임 긴장
└── 출판 어려움 (분량)
```

## 철학적 기반

### 실용주의 (Pragmatism)

```
핵심 주장:
├── 연구 질문이 방법 결정
├── 실용적 결과 중시
├── 패러다임 전쟁 거부
├── "무엇이 작동하는가?"
└── 다원주의적 입장

대표 학자:
├── Dewey
├── James
├── Peirce
└── Rorty
```

### 다중 패러다임 입장

```
옵션:

1. A-paradigmatic
   └── 패러다임 논의 무시

2. 단일 패러다임
   └── 하나의 패러다임으로 통합

3. 다중 패러다임
   └── 각 방법에 다른 패러다임

4. 변증법적 접근
   └── 패러다임 간 대화
```

## 설계 유형

### 1. 수렴적 설계 (Convergent Design)

```
구조:
QUAN        +        QUAL
  ↓                    ↓
분석                  분석
  ↓                    ↓
  └───── 통합 ─────┘
           ↓
        해석

특징:
├── 동시 수집
├── 동등한 가중치
├── 별도 분석
├── 결과 통합
└── 삼각화 목적

적합한 상황:
├── 결과 확증 필요
├── 현상의 다면적 이해
├── 시간 제약
└── 팀 연구
```

### 2. 설명적 순차 설계 (Explanatory Sequential)

```
구조:
QUAN → quan 분석 → QUAL → qual 분석 → 해석
        (결과가 QUAL 설계 정보 제공)

특징:
├── 양적 우선
├── 질적이 양적 설명
├── 순차적 수집
├── 연결 단계 중요
└── 양적 가중치 높음

적합한 상황:
├── 양적 결과 설명 필요
├── 예상외 결과 탐구
├── 참여자 관점 추가
└── 이상치 이해
```

### 3. 탐색적 순차 설계 (Exploratory Sequential)

```
구조:
QUAL → qual 분석 → QUAN → quan 분석 → 해석
        (결과가 QUAN 설계 정보 제공)

특징:
├── 질적 우선
├── 도구 개발 가능
├── 질적 결과 검증
├── 일반화 추구
└── 질적 → 양적 연결

적합한 상황:
├── 측정 도구 부재
├── 변수 탐색 필요
├── 이론 개발 후 검증
└── 현상 이해 → 일반화
```

### 4. 내재적 설계 (Embedded Design)

```
구조:
┌─────────────────────────┐
│   QUAN (또는 QUAL)      │
│  ┌─────────────────┐    │
│  │  qual (보조)    │    │
│  └─────────────────┘    │
└─────────────────────────┘

특징:
├── 하나의 방법이 주
├── 다른 방법이 보조
├── 비대칭 가중치
├── 보조가 주를 지원
└── 실험/사례연구에 내재

적합한 상황:
├── 실험에 과정 질문 추가
├── 사례연구에 설문 추가
├── 개입 연구
└── 효과 + 과정 동시 파악
```

### 5. 변형적 설계 (Transformative Design)

```
구조:
[사회정의/이론적 렌즈]
          ↓
    QUAN + QUAL
          ↓
   사회 변화 목표

특징:
├── 이론적 프레임워크 주도
├── 권한 부여 목적
├── 참여적 접근
├── 사회정의 지향
└── 변화 추구

적합한 상황:
├── 주변화된 집단 연구
├── 페미니스트 연구
├── 비판적 연구
└── 참여적 행동 연구
```

## 통합 (Integration)

### 통합의 수준

```
1. 설계 수준
   └── 연구 질문에 기반한 설계

2. 방법 수준
   └── 데이터 수집/분석 연결

3. 해석 수준
   └── 결과 통합 해석

4. 보고 수준
   └── 통합적 글쓰기
```

### 통합 기법

#### 1. Merging (합병)
```
방법:
├── 양적/질적 결과 나란히 배치
├── 공동 디스플레이 (Joint Display)
├── 수렴/발산 확인
└── 통합 해석

Joint Display 예시:
┌────────────┬────────────┬────────────┐
│ 양적 결과  │ 질적 결과  │ 통합 해석  │
├────────────┼────────────┼────────────┤
│ X-Y 정관계 │ "X하면 Y"  │ 확증됨     │
│ r=.45      │ 경험 묘사  │ 메커니즘   │
├────────────┼────────────┼────────────┤
│ Z 비유의   │ Z 중요성   │ 불일치     │
│ p>.05      │ 강조됨     │ 추가 탐구  │
└────────────┴────────────┴────────────┘
```

#### 2. Connecting (연결)
```
방법:
├── 한 방법 결과 → 다른 방법 정보 제공
├── 참여자 선정 연결
├── 도구 개발 연결
└── 순차적 빌딩

예시:
1. 질적 인터뷰 → 주제 도출
2. 주제 → 설문 문항 개발
3. 설문 조사 실시
4. 양적 결과 해석에 질적 통찰 활용
```

#### 3. Building (구축)
```
방법:
├── 한 방법이 다른 방법 구축
├── 도구 개발
├── 프레임워크 개발
└── 이론 검증

과정:
질적 탐색 → 개념 모델 → 가설 → 양적 검증
```

#### 4. Embedding (내재)
```
방법:
├── 보조 방법 내재
├── 주 방법 지원
├── 과정 이해 추가
└── 맥락 정보 추가
```

## 표본 추출

### Mixed Methods 표본 전략

```
1. 동일 표본 (Same sample)
   ├── 동일 참여자에게 양적+질적
   ├── 통합 용이
   └── 부담 증가

2. 중첩 표본 (Nested sample)
   ├── 양적 표본에서 질적 하위 표본
   ├── 일반화 + 심층 이해
   └── 선정 기준 명확히

3. 병렬 표본 (Parallel sample)
   ├── 동일 모집단, 다른 표본
   ├── 각 방법에 적합한 표본
   └── 통합 어려움

4. 다층 표본 (Multilevel sample)
   ├── 다른 수준에서 다른 방법
   ├── 예: 조직 수준 질적, 개인 수준 양적
   └── 다수준 분석
```

### 질적 하위 표본 선정

```
기준:

1. 극단 사례 (Extreme Cases)
   └── 양적에서 높거나 낮은 사례

2. 전형적 사례 (Typical Cases)
   └── 평균 근처 대표 사례

3. 확증/불확증 (Confirming/Disconfirming)
   └── 양적 결과 확인/반박 사례

4. 무작위 선정
   └── 대표성 추구

5. 이론적 표본추출
   └── 이론 개발에 필요한 사례
```

## 품질 기준

### 통합 품질

```
O'Cathain (2010) 기준:

1. 계획적 통합 (Planned)
   └── 설계 단계에서 통합 계획

2. 투명한 통합 (Transparent)
   └── 통합 과정 명시

3. 정당화된 통합 (Justified)
   └── 왜 통합했는지 설명

4. 통합의 성과 (Yield)
   └── 통합으로 얻은 것 명시
```

### Legitimation 유형 (Onwuegbuzie & Johnson)

```
1. 표본 통합 적법성
   └── 표본 관계 정당화

2. 내부자-외부자 적법성
   └── 에믹/에틱 균형

3. 약점 최소화 적법성
   └── 각 방법 약점 보완

4. 순차적 적법성
   └── 순차 연결 타당성

5. 전환 적법성
   └── 데이터 변환 정당성

6. 패러다임 혼합 적법성
   └── 패러다임 결합 정당화

7. 정치적 적법성
   └── 이해관계자 관점 균형
```

## 글쓰기

### 구조 옵션

```
1. 통합적 구조
   └── 양적/질적 함께 제시

2. 순차적 구조
   └── 양적 먼저, 질적 다음 (또는 반대)

3. 샌드위치 구조
   └── 통합-양적-질적-통합

4. 병렬 구조
   └── 연구질문별로 양적+질적 함께
```

### 통합 결과 제시

```
Joint Display 활용:

표 형식:
├── 양적 결과 열
├── 질적 결과 열
├── 통합 해석 열
└── 함의 열

시각적 모델:
├── 수렴/발산 표시
├── 보완 관계 표시
├── 통합 결론 제시
└── 메타 추론 명시
```
