# Academy of Management Review (AMR)
## 컨셉논문 투고 가이드

## 저널 개요

- **Impact Factor**: ~12.0 (경영학 Top Tier)
- **게재율**: ~8%
- **논문 유형**: 이론 논문 전문 (실증 X)
- **분량**: 약 12,000 단어

## AMR가 찾는 것

### 이론적 기여의 유형
1. **새로운 개념 도입** (Introducing a new construct)
2. **기존 이론 확장** (Extending existing theory)
3. **기존 이론 통합** (Integrating theories)
4. **기존 가정 도전** (Challenging assumptions)
5. **메커니즘 설명** (Explaining mechanisms)

### "So What?" 테스트
> "만약 이 논문이 게재되지 않는다면, 학계가 무엇을 모르게 되는가?"

## 논문 구조

### 1. Introduction (약 1,500 단어)

#### Hook: 관심 유발
```
좋은 예:
"매년 수백억 달러가 리더십 개발에 투자되지만,
대부분의 프로그램은 실패한다. 왜일까?"

피해야 할 예:
"리더십은 중요하다. 본 논문은 리더십에 대해 논의한다."
```

#### Gap: 문헌의 공백
```
- 기존 연구가 다루지 않은 것
- 기존 이론의 한계
- 모순되는 발견들
- 새로운 현상
```

#### So What: 왜 중요한가
```
- 이론적 중요성
- 실무적 중요성
- 시의성
```

#### Contribution: 기여 명시
```
"본 논문은 세 가지 기여를 한다:
첫째, ... 개념을 도입한다.
둘째, ... 이론을 확장한다.
셋째, ... 에 대한 새로운 관점을 제시한다."
```

### 2. Literature Review (약 3,000 단어)

#### 목적
- 관련 연구 종합
- 이론적 배경 제공
- Gap 논증

#### 구조 옵션
```
옵션 1: 주제별 조직
- 주제 A에 대한 연구
- 주제 B에 대한 연구
- 통합 및 비판

옵션 2: 이론별 조직
- 이론 1의 관점
- 이론 2의 관점
- 한계와 통합 필요성

옵션 3: 시간순 조직
- 초기 연구
- 발전 과정
- 현재 상태와 Gap
```

### 3. Theory Development (약 5,000 단어)

#### 개념 정의
```
"[개념명]을 '... '로 정의한다.

이 정의는 세 가지 핵심 요소를 포함한다:
(1) ...
(2) ...
(3) ..."
```

#### 명제 개발
```
Proposition 1: [선행요인]이 높을수록, [개념]이 증가한다.

[논리적 논증]

이 관계가 발생하는 이유는...
첫째, ...
둘째, ...
따라서, ...
```

#### 경계 조건
```
"이 관계는 [조건]에서 특히 강하게 나타난다.

왜냐하면 [조건]은 [메커니즘]을 강화하기 때문이다."
```

### 4. Discussion (약 2,500 단어)

#### 이론적 함의
```
"본 연구는 [분야] 문헌에 여러 함의를 제공한다.

첫째, [기존 이론]을 [방향]으로 확장한다...
둘째, [논쟁]에 대해 [관점]을 제시한다...
셋째, [새로운 연구 방향]을 열어준다..."
```

#### 실무적 함의
```
"실무자들에게 본 연구는 다음을 시사한다..."
```

#### 한계와 향후 연구
```
"본 연구의 한계를 인정하며, 이는 향후 연구 기회를 제시한다.

첫째, ... 한계가 있다. 향후 연구에서는...
둘째, ... 을 다루지 못했다. 이는..."
```

## AMR 리뷰 기준

### 심사 기준 (공식)
1. **이론적 기여**: 얼마나 새롭고 중요한가?
2. **논리적 엄격성**: 논증이 타당한가?
3. **개념적 명확성**: 개념이 명확히 정의되었는가?
4. **문헌 통합**: 기존 연구를 적절히 다루었는가?
5. **글쓰기 품질**: 명확하고 읽기 쉬운가?

### 흔한 리젝션 사유

#### 1. "So What?" (기여 불명확)
```
문제: "기여가 무엇인지 모르겠다"
대응: Introduction에서 명확히 기여 명시
      Discussion에서 함의 강화
```

#### 2. "Old Wine in New Bottle" (신규성 부족)
```
문제: "이미 알려진 것의 재포장"
대응: 기존 개념과의 명확한 차별화
      새로운 통찰의 원천 명시
```

#### 3. "Logic Gap" (논리적 비약)
```
문제: "A에서 B로의 논리적 연결이 부족"
대응: 중간 논리 단계 추가
      "왜냐하면" 구문 활용
```

#### 4. "Missing Literature" (문헌 누락)
```
문제: "핵심 문헌을 다루지 않았다"
대응: 체계적 문헌 검색
      핵심 저자/저널 확인
```

## 성공적인 AMR 논문 특징

### 좋은 개념 논문의 특징
1. **명확한 핵심 개념**: 한 문장으로 설명 가능
2. **논리적 흐름**: 각 섹션이 자연스럽게 연결
3. **적절한 범위**: 너무 넓지도 좁지도 않음
4. **설득력 있는 논증**: "왜"가 명확히 설명됨
5. **우아한 글쓰기**: 읽기 쉽고 기억에 남음

### 명작 논문 예시
- Weick (1995) - Sensemaking
- Eisenhardt & Martin (2000) - Dynamic Capabilities
- Edmondson (1999) - Psychological Safety
- March (1991) - Exploration vs Exploitation
