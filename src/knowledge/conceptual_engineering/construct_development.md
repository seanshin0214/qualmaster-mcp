# 구성개념 개발 (Construct Development)
## 신급 개념화 기법

---

## 구성개념이란?

### 정의
```
구성개념 (Construct):
├── 직접 관찰 불가능한 추상적 개념
├── 이론적으로 정의된 개념
├── 경험적으로 측정 가능해야 함
├── 다른 개념과 관계를 가짐
└── 현상을 설명하는 데 사용

예시:
├── 심리적 안전감 (Psychological Safety)
├── 조직 정체성 (Organizational Identity)
├── 센스메이킹 (Sensemaking)
├── 동적 역량 (Dynamic Capabilities)
└── 직무 배태성 (Job Embeddedness)
```

### 좋은 구성개념의 조건

```
MacKenzie et al. (2011):

1. 명확성 (Clarity)
   ├── 정확한 정의
   └── 모호함 없음

2. 간결성 (Parsimony)
   ├── 최소한의 속성
   └── 불필요한 복잡성 배제

3. 일관성 (Coherence)
   ├── 논리적 일관성
   └── 내적 모순 없음

4. 변별성 (Distinctiveness)
   ├── 기존 개념과 구분
   └── 고유한 의미 영역

5. 적용성 (Applicability)
   ├── 측정 가능
   └── 실증 연구 가능
```

## 구성개념 개발 프로세스

### 1단계: 현상 관찰과 문제 인식

```
시작점:
├── 현장 관찰에서 포착된 패턴
├── 기존 개념으로 설명 안 되는 현상
├── 이론-실제 괴리
├── 문헌의 모순
└── 직관적 통찰

핵심 질문:
"왜 기존 개념으로 이것을 설명할 수 없는가?"
"새로운 개념이 필요한 이유는 무엇인가?"
```

### 2단계: 개념적 정의 (Conceptual Definition)

```
정의의 구성요소:

1. 속 (Genus): 상위 범주
   "X는 일종의 [상위 개념]이다"

2. 종차 (Differentia): 구별 특성
   "다른 [상위 개념]과 달리, X는..."

3. 필수 속성: 핵심 특징
   "X의 핵심 특성은..."

예시 - 심리적 안전감:

속: 팀 분위기(team climate)의 일종
종차: 대인 위험 감수에 대한 믿음
필수 속성:
├── 공유된 믿음
├── 대인 위험에 초점
├── 팀 수준
└── 발언 가능성에 대한 인식
```

### 3단계: 차원/속성 식별 (Dimensions/Attributes)

```
차원 구조 옵션:

1. 단일차원 (Unidimensional)
   ├── 하나의 연속선
   └── 예: 직무 만족

2. 다차원 잠재 (Multidimensional Latent)
   ├── 여러 차원이 상위 구성개념 형성
   └── 예: 조직 공정성 (분배, 절차, 상호작용)

3. 형성적 다차원 (Multidimensional Formative)
   ├── 여러 차원이 구성개념을 구성
   └── 예: SES (소득 + 교육 + 직업)

4. 유형론 (Typology)
   ├── 질적으로 다른 유형
   └── 예: 리더십 스타일
```

### 4단계: 노몰로지적 네트워크 (Nomological Network)

```
네트워크 구성:

         [선행요인]
              ↓
    [관련 구성개념] ←→ [핵심 구성개념] ←→ [관련 구성개념]
              ↓
         [결과변수]

정의해야 할 것:
├── 선행요인 (Antecedents)
├── 결과변수 (Outcomes)
├── 상관관계 (Correlates)
├── 조절변수 (Moderators)
└── 매개변수 (Mediators)
```

### 5단계: 기존 개념과 구분 (Discriminant Validity)

```
구분해야 할 개념 식별:

1. 상위 개념 (Broader Constructs)
   "X는 Y의 하위 개념이지만,
    Y 전체와는 다르다"

2. 유사 개념 (Similar Constructs)
   "X는 Z와 관련되지만,
    다음 점에서 구별된다..."

3. 반대 개념 (Opposite Constructs)
   "X는 W의 부재가 아니다"

표로 정리:
┌────────────┬─────────────┬─────────────┐
│ 개념       │ 유사점      │ 차이점      │
├────────────┼─────────────┼─────────────┤
│ 기존개념 A │ ...         │ ...         │
│ 기존개념 B │ ...         │ ...         │
│ 기존개념 C │ ...         │ ...         │
└────────────┴─────────────┴─────────────┘
```

## 정의 작성 기법

### 좋은 정의의 특징

```
1. 명확성 (Clear)
   ├── 모호한 용어 배제
   └── 구체적 언어

2. 간결성 (Concise)
   ├── 필수 요소만 포함
   └── 장황하지 않음

3. 완전성 (Complete)
   ├── 핵심 속성 포함
   └── 빠진 것 없음

4. 구분가능성 (Distinguishing)
   ├── 다른 개념과 구별
   └── 고유한 의미 영역
```

### 정의 작성 공식

```
공식 1: 속+종차 정의
"[구성개념]은 [상위범주]로서,
 [구별되는 특성]을 가진다."

공식 2: 속성 나열 정의
"[구성개념]은 다음을 포함한다:
 (a) [속성1], (b) [속성2], (c) [속성3]."

공식 3: 기능적 정의
"[구성개념]은 [맥락]에서
 [행위자]가 [행동/상태]하는 것이다."
```

### 예시 분석

```
심리적 안전감 (Edmondson, 1999):

정의: "팀 심리적 안전감은 팀이 대인 위험 감수에
      안전하다는 공유된 믿음이다."

분석:
├── 속: 공유된 믿음
├── 맥락: 팀
├── 핵심 특성: 대인 위험 감수
├── 수준: 팀 수준
└── 유형: 인지적 상태

강점:
├── 간결함
├── 핵심 특성 명확
├── 측정 가능
└── 기존 개념과 구별
```

## 차원 개발

### 차원 도출 방법

```
1. 이론적 도출 (Theoretical Derivation)
   ├── 기존 이론에서
   ├── 논리적 분석에서
   └── 연역적 접근

2. 경험적 도출 (Empirical Derivation)
   ├── 데이터에서
   ├── 인터뷰/관찰에서
   └── 귀납적 접근

3. 혼합 접근 (Mixed Approach)
   ├── 이론 + 경험
   ├── 반복적 정교화
   └── 가장 권장됨
```

### 차원 구조 결정

```
질문:
1. 차원들이 상관관계가 있는가?
   → 높으면: 반영적 모델 고려
   → 낮으면: 형성적 모델 고려

2. 차원 제거 시 의미가 변하는가?
   → 변하면: 형성적
   → 안 변하면: 반영적

3. 차원들의 선행/결과변수가 같은가?
   → 같으면: 반영적
   → 다르면: 형성적
```

### 차별화 논증 구조

```
"[새 구성개념]은 [기존 개념]과 관련되지만,
 중요한 차이가 있다.

첫째, [차이점 1].
[기존 개념]이 [특성 A]에 초점을 맞추는 반면,
[새 구성개념]은 [특성 B]를 강조한다.

둘째, [차이점 2].
[기존 개념]이 [수준/맥락 C]에서 주로 다루어진 반면,
[새 구성개념]은 [수준/맥락 D]에 적용된다.

셋째, [차이점 3].
[기존 개념]의 선행요인이 [X]인 반면,
[새 구성개념]은 [Y]에 의해 영향받는다."
```

## 명명 (Naming)

### 좋은 이름의 조건

```
1. 직관적 (Intuitive)
   ├── 의미가 명확히 전달
   └── 기억하기 쉬움

2. 구별적 (Distinctive)
   ├── 기존 용어와 혼동 없음
   └── 고유한 영역

3. 적절한 (Appropriate)
   ├── 개념 본질 반영
   └── 과장/축소 없음

4. 학문적 (Scholarly)
   ├── 학술적으로 수용 가능
   └── 전문성 있는 용어
```

### 명명 전략

```
1. 기존 용어 재정의
   예: "Safety" → "Psychological Safety"

2. 용어 조합
   예: "Sense" + "Making" → "Sensemaking"

3. 메타포 활용
   예: "Organizational DNA"

4. 라틴어/그리스어 활용
   예: "Ambidexterity"

5. 기술적 용어
   예: "Job Embeddedness"
```

## 질적연구에서의 구성개념 개발

### Gioia 방법론

```
1차 개념 (1st Order Concepts):
├── 참여자 언어
├── 인비보 코드
├── 경험적 기술
└── 다수 (수십 개)

2차 주제 (2nd Order Themes):
├── 연구자 언어
├── 이론적 개념화
├── 추상화
└── 중간 수준 (수~십여 개)

총괄 차원 (Aggregate Dimensions):
├── 이론적 범주
├── 높은 추상화
├── 핵심 구성개념
└── 소수 (3-7개)
```

### 데이터 구조 (Data Structure)

```
1차 개념          2차 주제          총괄 차원
────────────────────────────────────────────
• 인용문 1 ─┐
• 인용문 2 ─┼─→ [주제 A] ─┐
• 인용문 3 ─┘              │
                           ├─→ [차원 I]
• 인용문 4 ─┐              │
• 인용문 5 ─┼─→ [주제 B] ─┘
• 인용문 6 ─┘

• 인용문 7 ─┐
• 인용문 8 ─┼─→ [주제 C] ─┐
• 인용문 9 ─┘              │
                           ├─→ [차원 II]
• 인용문 10 ─┐             │
• 인용문 11 ─┼─→ [주제 D] ─┘
• 인용문 12 ─┘
```

## 흔한 실수와 대응

### 1. 정의가 너무 넓음
```
문제: 무엇이든 포함될 수 있음
대응: 경계 명확히, 포함/비포함 예시
```

### 2. 정의가 너무 좁음
```
문제: 너무 제한적, 유용성 저하
대응: 핵심 본질에 집중, 불필요한 제한 제거
```

### 3. 기존 개념과 중복
```
문제: "old wine in new bottle"
대응: 명확한 구분표, 차별화 논증 강화
```

### 4. 차원이 불분명
```
문제: 차원 간 경계 모호
대응: 각 차원 독립적 정의, 중복 제거
```

### 5. 측정 불가능
```
문제: 너무 추상적이어서 측정 어려움
대응: 조작적 정의 가능성 확인, 지표 예시
```
