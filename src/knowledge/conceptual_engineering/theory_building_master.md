## 이론이란 무엇인가?

### Whetten (1989)의 정의
```
완전한 이론의 4요소:

WHAT: 무엇이 포함되는가?
├── 핵심 구성개념 (Constructs)
├── 변수 (Variables)
└── 경계 조건 (Boundary Conditions)

HOW: 어떻게 연결되는가?
├── 구성개념 간 관계
├── 인과 메커니즘
└── 과정과 순서

WHY: 왜 그러한가?
├── 논리적 근거
├── 이론적 메커니즘
└── 설명의 논리

WHO/WHERE/WHEN: 경계 조건
├── 적용 범위
├── 맥락 조건
└── 시간적 제약
```

### Sutton & Staw (1995): 이론이 아닌 것
```
이론이 아닌 것들:

1. 참고문헌 ≠ 이론
   "선행연구가 보여주듯이..."
   → 왜 그런지 설명이 없으면 이론 아님

2. 데이터 ≠ 이론
   "설문 결과 X와 Y는 상관관계..."
   → 왜 관계가 있는지 설명 필요

3. 변수 목록 ≠ 이론
   "리더십, 동기, 성과가 중요하다"
   → 어떻게 연결되는지 설명 필요

4. 도표/그림 ≠ 이론
   박스와 화살표
   → 왜 그렇게 연결되는지 설명 필요

5. 가설 ≠ 이론
   "X가 높을수록 Y가 높다"
   → 왜 그런지 논증이 이론
```

## 이론적 기여의 유형

### Corley & Gioia (2011): 2×2 프레임워크
```
           독창성 (Originality)
           낮음          높음
          ┌──────────┬──────────┐
    높음  │ 점진적    │ 혁명적   │
유         │ 확장      │ 창조     │
용         │ (많은     │ (드문    │
성         │  논문)    │  명작)   │
          ├──────────┼──────────┤
    낮음  │ 사소한    │ 과학적   │
          │ 변형      │ 호기심   │
          │ (리젝션)  │ (틈새)   │
          └──────────┴──────────┘
```

### 기여 유형 상세

#### 1. 새로운 구성개념 도입 (New Construct)
```
예시:
├── Edmondson: 심리적 안전감
├── Weick: 센스메이킹
├── Dutton: 에너지 관계
└── Spreitzer: 심리적 임파워먼트

요건:
├── 기존 개념과 명확히 구분
├── 이론적 필요성 논증
├── 정확한 정의
├── 명확한 차원/속성
└── 측정 가능성 (또는 향후 가능성)
```

#### 2. 기존 이론 확장 (Theory Extension)
```
방식:
├── 새로운 선행요인 추가
├── 새로운 결과변수 추가
├── 조절변수 발견
├── 매개 메커니즘 규명
└── 경계 조건 명시

예시:
"기존 X 이론은 Y 맥락에서 어떻게 작동하는가?"
"Z 조건에서 X-Y 관계는 어떻게 달라지는가?"
```

#### 3. 기존 이론 통합 (Theory Integration)
```
방식:
├── 분절된 이론들의 연결
├── 상위 프레임워크 제안
├── 모순 해소
└── 종합적 모델 개발

주의:
├── 통합의 필요성 논증 필수
├── 단순 나열 아님
├── 새로운 통찰 필요
└── 1+1 > 2 보여주기
```

#### 4. 기존 가정 도전 (Challenging Assumptions)
```
방식:
├── 당연시된 가정 식별
├── 가정의 문제점 논증
├── 대안적 관점 제시
└── 새로운 연구 방향 제안

예시:
"기존 연구는 X를 가정하지만,
 실제로는 ~X일 수 있다"
```

## 이론 구축 프로세스

### 1단계: 현상 포착 (Phenomenon Catching)

```
좋은 현상의 특징:
├── 흥미로움 (Interesting)
├── 중요성 (Significant)
├── 설명되지 않음 (Unexplained)
├── 역설적 (Paradoxical)
└── 시의성 (Timely)

발견 방법:
├── 실무 관찰
├── 뉴스/트렌드
├── 개인 경험
├── 문헌 모순
├── 이론-현실 괴리
└── 이상 현상 (Anomaly)
```

### 2단계: 개념화 (Conceptualization)

```
과정:
관찰 → 추상화 → 개념 → 정의

개념화의 핵심:
├── 본질 포착
├── 적절한 추상 수준
├── 명확한 경계
├── 기존 개념과 구분
└── 이론적 유용성
```

### 3단계: 관계 설정 (Relationship Building)

```
관계의 유형:
├── 선행-결과 (Antecedent-Outcome)
├── 조절 (Moderation)
├── 매개 (Mediation)
├── 과정 (Process)
└── 구성 (Constitution)

논증의 핵심 = "WHY"
├── 왜 X가 Y를 일으키는가?
├── 어떤 메커니즘으로?
├── 어떤 조건에서?
└── 논리적 연결고리
```

### 4단계: 경계 설정 (Boundary Setting)

```
경계 조건:
├── WHO: 누구에게 적용?
├── WHERE: 어떤 맥락에서?
├── WHEN: 어떤 시점에?
└── 적용 범위와 한계

예시:
"이 이론은 A 조건에서 강하게,
 B 조건에서 약하게 적용된다"
```

## 논증 구조 (Argumentation)

### Toulmin 모델 적용

```
[Data/Claim]
    ↓
[Warrant: 왜 그런가?]
    ↓
[Backing: 근거 지지]
    ↓
[Qualifier: 조건/한정]
    ↓
[Rebuttal: 반론 대응]
    ↓
[Conclusion]
```

### 명제 개발 구조

```
좋은 명제 구조:

Proposition X: [관계 진술]

논증:
1. 전제 1 (이론적 기반)
2. 전제 2 (논리적 연결)
3. 전제 3 (메커니즘 설명)
4. 따라서 (결론)

경계 조건:
"이 관계는 Z 조건에서 특히 강하게 나타난다.
 왜냐하면..."
```

### 논증 강화 전략

```
1. 메커니즘 명시
   "X가 Y에 영향을 미치는 이유는
    M 메커니즘을 통해서이다"

2. 대안 설명 배제
   "A 설명이 아닌 이유는..."
   "B보다 C가 더 타당한 이유는..."

3. 다층 논증
   미시(개인) + 거시(조직/사회) 연결

4. 시간적 논리
   "먼저 A가 일어나고, 그 다음 B가..."

5. 반직관적 통찰
   "일반적으로 X라고 생각하지만,
    실제로는 ~X이다. 왜냐하면..."
```

## 명제 vs 가설

### 구분
```
명제 (Proposition):
├── 개념 수준
├── 이론적 진술
├── 직접 검증 어려움
├── 개념 논문에서 사용
└── 예: "심리적 안전감은 학습 행동을 촉진한다"

가설 (Hypothesis):
├── 변수 수준
├── 경험적 진술
├── 직접 검증 가능
├── 실증 논문에서 사용
└── 예: "심리적 안전감 점수가 높은 팀은
        학습 행동 점수가 높다"
```

### 명제 작성법

```
형식:
Proposition 1: [개념 A]가 [조건]에서 
              [개념 B]에 [방향] 영향을 미친다.

예시:
Proposition 1: 리더의 겸손함은 
              팀 심리적 안전감을 증가시키며,
              이는 팀 학습 행동을 촉진한다.

Proposition 1a: 이 관계는 팀 다양성이 
               높을 때 더 강하게 나타난다.
```

## 시각적 모델 개발

### 이론적 모델의 유형

```
1. 프로세스 모델 (Process Model)
   시간 흐름에 따른 단계
   [단계1] → [단계2] → [단계3]

2. 분산 모델 (Variance Model)
   변수 간 관계
   [X] ─→ [Y]
         ↑
        [M]

3. 유형론 (Typology)
   차원에 따른 분류
   2×2 매트릭스

4. 다층 모델 (Multi-level Model)
   분석 수준 통합
   [거시] ─→ [개인] ─→ [거시]
```

### 시각화 원칙

```
DO:
├── 핵심 관계만 포함
├── 깔끔하고 읽기 쉽게
├── 변수/개념 명확히 레이블
├── 관계 방향 표시
├── 본문에서 충분히 설명

DON'T:
├── 너무 복잡하게
├── 모든 것을 넣으려고
├── 설명 없는 도표
├── 모호한 연결선
└── 그림만으로 이론 대체
```

## 글쓰기 전략

### 서론 구조 (Hook-Gap-So What-Contribution)

```
문단 1: Hook
"[충격적 사실/역설/질문]으로 시작"

문단 2-3: 문헌 검토 & Gap
"기존 연구는 A를 밝혔지만, B는 미해결"

문단 4: So What
"이 gap이 중요한 이유는..."

문단 5: Contribution
"본 논문은 세 가지 기여를 한다:
 첫째, ... 둘째, ... 셋째, ..."
```

### 이론 섹션 구조

```
1. 개념 정의
   "X를 '...'로 정의한다"

2. 이론적 배경
   "이 개념은 Y 이론에 기반하며..."

3. 명제 개발
   "Proposition 1: ..."
   [논증 단락]

4. 경계 조건
   "이 관계는 Z 조건에서..."

5. 종합 모델
   [Figure 제시]
   [모델 설명]
```

### Discussion 구조

```
1. 요약
   "본 연구는..."

2. 이론적 함의
   "X 이론에 세 가지 기여:
    첫째, ... 둘째, ... 셋째, ..."

3. 실무적 함의
   "관리자들에게..."

4. 한계와 향후 연구
   "본 연구의 한계는..."

5. 결론
   "결론적으로..."
```

## 흔한 실수와 대응

### 1. "So What?" 문제
```
증상: 기여가 불명확
대응: 
├── 서론에서 기여 명시
├── "왜 중요한가" 반복 강조
├── 함의 구체적으로
└── 변화시키는 것 명시
```

### 2. "Old Wine in New Bottle"
```
증상: 새로운 것 없음
대응:
├── 기존 개념과 명확 구분
├── 새로운 통찰 명시
├── "왜 새로운 개념이 필요한가"
└── 기존 이론의 한계 논증
```

### 3. "Logic Gap"
```
증상: A에서 B로의 논리 비약
대응:
├── 중간 논리 단계 추가
├── "왜냐하면" 구문 사용
├── 각 단계 연결 명시
└── 대안 설명 배제
```

### 4. "Underdeveloped"
```
증상: 아이디어는 좋으나 덜 발전
대응:
├── 각 개념 상세 정의
├── 차원/속성 명시
├── 메커니즘 충분히
├── 경계 조건 구체화
└── 예시 풍부하게
```
