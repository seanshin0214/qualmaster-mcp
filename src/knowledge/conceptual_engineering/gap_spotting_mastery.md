# Gap Spotting 완전 정복
## 연구 공백 발견과 활용의 기술

---

## Gap이란 무엇인가?

### 정의
```
연구 Gap = 기존 연구가 다루지 않은 영역
         = 이론적 기여의 기회
         = 새로운 연구의 정당성

유형:
├── 이론적 공백
├── 경험적 공백
├── 방법론적 공백
├── 맥락적 공백
└── 시간적 공백
```

### Sandberg & Alvesson (2011): Gap-Spotting 비판

```
Gap-Spotting의 한계:
├── 지식 축적에는 기여
├── 하지만 영향력 있는 연구는 드묾
├── 기존 가정을 문제시하지 않음
└── 점진적 개선에 그침

더 강력한 대안: Problematization
├── 기존 가정에 도전
├── 새로운 연구 질문 생성
├── 패러다임 전환 가능
└── 높은 영향력
```

## Gap의 유형

### 1. Confusion Spotting (혼란 발견)

```
정의: 
기존 연구들 간의 모순/불일치 발견

예시:
"선행연구들은 X-Y 관계에 대해 상반된 결과를 보여준다.
 A(2020)는 정(+)의 관계를, B(2021)는 부(-)의 관계를,
 C(2022)는 무관계를 보고했다."

활용:
├── 왜 불일치하는지 설명
├── 조절변수 제안
├── 통합 프레임워크 개발
└── 메타분석 필요성 논증
```

### 2. Neglect Spotting (무시 발견)

```
정의:
연구되지 않은 영역, 변수, 관계 식별

예시:
"기존 연구는 X의 선행요인에 집중했으나,
 결과변수에 대한 연구는 부족하다."

"Y 현상은 A 맥락에서 주로 연구되었으나,
 B 맥락에서의 적용은 거의 없다."

활용:
├── 새로운 변수/관계 탐구
├── 새로운 맥락 적용
├── 무시된 집단 연구
└── 새로운 방법론 적용
```

### 3. Application Spotting (적용 발견)

```
정의:
기존 이론의 새로운 맥락 적용 가능성

예시:
"X 이론은 영리 조직에서 발전했으나,
 비영리/공공 조직에서의 적용은 제한적이다."

"개인 수준의 Y 이론을
 팀/조직 수준으로 확장할 필요가 있다."

활용:
├── 경계 조건 탐구
├── 이론 수정/확장
├── 다른 분야로 확대
└── 다른 수준으로 확장
```

### 4. Under-development Spotting (저발전 발견)

```
정의:
개념/이론이 충분히 발전되지 않음

예시:
"X 개념의 차원 구조가 불명확하다."
"Y 이론의 경계 조건이 명시되지 않았다."
"Z 관계의 매개 메커니즘이 규명되지 않았다."

활용:
├── 개념 정교화
├── 차원 구조 규명
├── 메커니즘 규명
└── 경계 조건 명시
```

## Problematization: Gap을 넘어서

### Alvesson & Sandberg (2011)

```
Problematization = 기존 가정에 도전

기존 가정 유형:

1. In-house Assumptions
   ├── 특정 연구 전통 내 가정
   └── 예: 리더십 연구에서 "리더는 팔로워에게 영향"

2. Root Metaphor Assumptions
   ├── 기저 은유
   └── 예: "조직은 기계이다"

3. Paradigm Assumptions
   ├── 패러다임 수준 가정
   └── 예: 실증주의 vs 해석주의

4. Ideology Assumptions
   ├── 이데올로기적 가정
   └── 예: 효율성이 최고 가치

5. Field Assumptions
   ├── 학문 분야 전체의 가정
   └── 예: "경영은 과학이다"
```

### Problematization 기법

```
1. 가정 식별
   "기존 연구가 당연시하는 것은?"

2. 가정 명료화
   "이 가정의 구체적 내용은?"

3. 가정 평가
   "이 가정은 타당한가?"

4. 대안 개발
   "다른 가정을 하면 어떨까?"

5. 새 연구 질문
   "대안적 가정에서 어떤 질문이?"
```

## Gap 발견 전략

### 1. 문헌 검토 전략

```
체계적 검토:
├── 핵심 저널 최근 10년
├── 인용 많은 논문
├── 리뷰 논문
├── 메타분석
└── 미래 연구 제안

질문:
├── 무엇이 연구되었나?
├── 무엇이 연구되지 않았나?
├── 어떤 모순이 있나?
├── 어떤 가정이 있나?
└── 무엇이 당연시되나?
```

### 2. 문헌 맵핑

```
X축: 이론적 관점
Y축: 연구 맥락/대상

    │ 이론 A │ 이론 B │ 이론 C
────┼────────┼────────┼────────
맥락1│  많음  │  약간  │  없음  ← Gap!
맥락2│  약간  │  많음  │  없음  ← Gap!
맥락3│  없음  │  없음  │  없음  ← Gap!
      ↑
     Gap!
```

### 3. 리뷰 논문 활용

```
활용 위치:
├── Introduction의 Gap
├── Future Research 섹션
├── Limitations 섹션
└── Research Agenda

주의:
├── 리뷰 저자의 관점 한계
├── 자신의 관점으로 재평가
├── 여러 리뷰 비교
└── 최신 리뷰 우선
```

### 4. 학회/세미나 활용

```
기회:
├── 미출판 최신 연구
├── 연구자 토론 청취
├── 네트워킹으로 정보
└── 트렌드 파악

질문:
├── 현재 뜨거운 주제는?
├── 해결되지 않은 문제는?
├── 논쟁 중인 이슈는?
└── 앞으로 유망한 방향은?
```

## Gap 표현 전략

### 서론에서 Gap 제시

```
구조:

[문단 1: 주제 중요성]
"X 현상은 조직에 중요한 영향을 미친다..."

[문단 2: 기존 연구 요약]
"기존 연구들은 X의 A, B, C 측면을 밝혀왔다..."

[문단 3: Gap 제시]
"그러나 기존 연구에는 세 가지 한계가 있다.
 첫째, ...가 연구되지 않았다.
 둘째, ...에 대한 이해가 부족하다.
 셋째, ...맥락에서의 적용이 제한적이다."

[문단 4: 본 연구의 대응]
"본 연구는 이러한 공백을 해결하기 위해..."
```

### Gap 표현 문장 패턴

```
무시 Gap:
├── "Despite the importance of X, little research has examined..."
├── "There is a paucity of research on..."
├── "This area remains under-explored..."
└── "Few studies have investigated..."

혼란 Gap:
├── "Prior research has produced inconsistent findings..."
├── "The relationship between X and Y remains unclear..."
├── "There is considerable disagreement about..."
└── "These contradictory findings suggest..."

적용 Gap:
├── "While X theory has been applied in A context, 
│    its relevance to B context is unknown..."
├── "The generalizability of these findings to... is unclear..."
└── "Whether these results extend to... remains to be seen..."

저발전 Gap:
├── "The mechanisms underlying... are not well understood..."
├── "The boundary conditions of... have not been specified..."
├── "The dimensionality of... remains unclear..."
└── "The concept of... is underdeveloped..."
```

## Gap과 Contribution 연결

### 논리적 연결

```
Gap                    →  Contribution
────────────────────────────────────────
연구되지 않음          →  새로운 발견 제공
모순적 결과           →  통합/조절변수 제안
새 맥락 미적용         →  이론 확장/경계 검증
개념 저발전           →  개념 정교화
메커니즘 불명확       →  메커니즘 규명
경계조건 불명시       →  경계조건 명시
```

### Gap 없이 기여하기

```
Problematization 접근:

Gap 대신:
├── 기존 가정의 문제 지적
├── 새로운 관점 제안
├── 패러다임 도전
└── 근본적 재고 촉구

예시:
"기존 연구는 X를 당연시해왔다.
 그러나 이 가정에는 문제가 있다.
 왜냐하면... 따라서 본 연구는
 대안적 관점을 제안한다."
```

## 흔한 실수와 대응

### 1. 사소한 Gap
```
문제: "연구된 적 없다"만으로는 부족
대응: 왜 연구가 필요한지 중요성 논증
     "So What?" 테스트 통과 필요
```

### 2. Gap 과장
```
문제: "전혀 연구되지 않았다"는 거짓일 수 있음
대응: 철저한 문헌 검토
     "제한적이다", "충분하지 않다" 표현
```

### 3. Gap과 Contribution 불일치
```
문제: Gap은 A인데 Contribution은 B
대응: 논리적 일관성 유지
     Gap이 본 연구로 어떻게 해결되는지 명시
```

### 4. 너무 많은 Gap
```
문제: 초점 분산, 기여 희석
대응: 2-3개 핵심 Gap에 집중
     관련 Gap들을 통합
```

## 분야별 Gap 특성

### 경영학/조직학
```
주요 Gap 유형:
├── 맥락 확장 (신흥국, 디지털 등)
├── 수준 간 연결 (개인-팀-조직)
├── 시간적 역동성 (과정, 변화)
└── 메커니즘 규명 (왜, 어떻게)
```

### 교육학
```
주요 Gap 유형:
├── 실무-이론 연결
├── 다양한 학습자 집단
├── 새로운 교육 환경 (온라인 등)
└── 장기 효과 연구
```

### 리더십/조직심리
```
주요 Gap 유형:
├── 어두운 면 (Dark side)
├── 팔로워 관점
├── 맥락 효과
└── 문화 간 차이
```
