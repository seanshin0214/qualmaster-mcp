# 주제분석 (Thematic Analysis)

## 개요

### 정의
주제분석은 데이터 내에서 반복되는 패턴(주제)을 식별하고, 분석하며, 보고하는 방법입니다.

### 특징
- **이론적 유연성**: 다양한 인식론적 입장에서 사용 가능
- **접근성**: 질적연구 입문자에게 적합
- **광범위한 적용**: 다양한 연구 질문에 적용

### Braun & Clarke (2006) 접근
가장 널리 인용되는 주제분석 프레임워크

## 6단계 절차

### 1단계: 데이터 친숙화 (Familiarization)

#### 활동
- 전사본 반복 읽기
- 면담이면 직접 전사 권장
- 초기 아이디어 메모

#### 질문
- "데이터가 전체적으로 무엇을 말하는가?"
- "어떤 패턴이 보이기 시작하는가?"

### 2단계: 초기 코드 생성 (Generating Initial Codes)

#### 코딩 접근
- **귀납적 코딩**: 데이터에서 코드 도출
- **연역적 코딩**: 이론/선행연구에서 코드 적용

#### 코딩 지침
- 체계적으로 전체 데이터셋 코딩
- 하나의 추출에 복수 코드 가능
- 맥락 유지

**예시**:
```
데이터: "처음 진단받았을 때 정말 무서웠어요.
        앞으로 어떻게 될지 전혀 모르겠더라고요."

코드: - 진단 시점의 공포
      - 미래 불확실성
      - 무지에서 오는 두려움
```

### 3단계: 주제 탐색 (Searching for Themes)

#### 과정
- 코드를 잠재적 주제로 분류
- 코드 간 관계 분석
- 주제 지도 또는 표 작성

#### 주제의 구조
```
상위 주제 (Overarching Theme)
    ├── 주제 1
    │   ├── 하위주제 1.1
    │   └── 하위주제 1.2
    └── 주제 2
        ├── 하위주제 2.1
        └── 하위주제 2.2
```

### 4단계: 주제 검토 (Reviewing Themes)

#### 2단계 검토

**1수준: 코드화된 추출과의 적합성**
- 주제가 코드화된 데이터와 일치하는가?
- 주제 지도 작성 (Thematic Map)

**2수준: 전체 데이터셋과의 적합성**
- 주제가 전체 데이터를 반영하는가?
- 추가 코딩 필요 여부 확인

#### 주제 조정
- 주제 병합 또는 분리
- 새로운 주제 추가
- 부적합한 주제 제거

### 5단계: 주제 정의와 명명 (Defining and Naming Themes)

#### 활동
- 각 주제의 본질 파악
- 주제가 포착하는 것 명확화
- 간결하고 의미 있는 이름 부여

#### 질문
- "이 주제의 핵심은 무엇인가?"
- "이 주제가 전체 이야기에서 어떤 역할을 하는가?"
- "주제 간의 관계는 무엇인가?"

### 6단계: 보고서 작성 (Producing the Report)

#### 글쓰기 목표
- 데이터의 복잡하고 풍부한 이야기 전달
- 주제의 타당성 논증
- 연구 질문에 대한 답변

#### 구조
```
1. 연구 맥락 소개
2. 분석 방법 설명
3. 주제별 결과 기술
   - 주제 설명
   - 데이터 추출 예시
   - 해석과 연결
4. 주제 간 관계 논의
5. 결론과 함의
```

## 주제의 유형

### 의미적 주제 vs 잠재적 주제

| 유형 | 의미적 주제 | 잠재적 주제 |
|------|-----------|-----------|
| 수준 | 표면적 의미 | 심층적 의미 |
| 분석 | 기술적 | 해석적 |
| 초점 | 말한 것 | 말 이면의 것 |

### 귀납적 vs 연역적 접근

**귀납적 (데이터 주도)**
- 데이터에서 주제 도출
- 이론적 선입견 최소화
- 상향식 접근

**연역적 (이론 주도)**
- 기존 이론/개념 적용
- 특정 관심 영역 초점
- 하향식 접근

## 품질 확보

### Braun & Clarke의 15개 체크리스트

**전사**
1. 적절한 수준으로 전사되었는가?

**코딩**
2. 각 데이터 항목에 동등한 주의를 기울였는가?
3. 코딩 과정이 철저하고 포괄적인가?
4. 주제가 데이터에 의해 뒷받침되는가?

**분석**
5. 데이터가 코드 및 주제와 대조되었는가?
6. 주제가 내적 일관성을 가지는가?
7. 데이터가 단순 요약이 아닌 분석되었는가?

**글쓰기**
8. 분석이 연구 질문에 답하는가?
9. 충분한 데이터 추출이 제시되었는가?

**전반**
10. 인식론적 입장이 일관되는가?

## 한계와 고려사항

### 방법론적 주의
- 단순한 주제 나열 피하기
- 분석적 서사 개발 필요
- 코드와 주제의 혼동 주의

### 비판
- 이론적 기반 부족 우려
- "anything goes" 위험
- 엄격성 문제

## 최신 발전: 반성적 주제분석

### Braun & Clarke (2019, 2021) 업데이트
- **반성적 (Reflexive)** 주제분석 강조
- 코드와 주제는 "발견"이 아닌 "생성"
- 연구자의 적극적 역할 인정
- 코딩 신뢰도 측정 지양
