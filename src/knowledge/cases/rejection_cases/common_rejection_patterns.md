# 질적연구 리젝션 패턴 완벽 분석 및 대응 전략

## 1. Editor Desk Rejection 패턴

### 패턴 1: "Not a Fit for This Journal"
```
리젝션 문구:
"While your work may have merit, it does not fit
the scope or aims of [Journal Name]..."

원인 분석:
├── 저널 Aims & Scope 미확인
├── 최근 게재 논문 트렌드 무시
├── 에디터 관심사 파악 실패
└── 분야/방법론 부적합

대응 전략:
□ 저널 최근 3년 질적연구 논문 분석
□ Editorial board 관심사 파악
□ Special issue 기회 활용
□ Cover letter에서 적합성 명시
```

### 패턴 2: "Contribution Not Clear"
```
리젝션 문구:
"The manuscript does not make a clear contribution
to existing theory/knowledge..."

원인 분석:
├── "So what?" 질문에 답 없음
├── 기술적(descriptive) 수준에 머무름
├── 기존 연구와 차별화 부족
└── Contribution statement 모호

대응 전략:
□ Introduction에 3개 contribution 명시
□ "We contribute to X by..." 문장 구조
□ What we knew vs What we now know 표
□ Discussion 첫 문단 contribution 요약
```

### 패턴 3: "Methodological Concerns"
```
리젝션 문구:
"The methodology lacks sufficient rigor to
support the claims made..."

원인 분석:
├── 분석 절차 불투명
├── 표집 논리 부재
├── 신뢰성 확보 방법 누락
└── 데이터와 해석 연결 불명확

대응 전략:
□ Gioia-style data structure 포함
□ 상세한 분석 절차 기술
□ Trustworthiness 섹션 추가
□ 대표적 증거(인용문) 제시
```

## 2. R1 Reject 패턴

### 패턴 4: "Interesting But Not Surprising"
```
리뷰어 코멘트:
"While the findings are interesting, they largely
confirm what we already know from existing research..."

원인 분석:
├── 기존 이론의 재확인에 그침
├── 새로운 통찰 부재
├── 반직관적(counterintuitive) 발견 없음
└── 맥락 특수성만 제시

대응 전략:
□ "What's surprising" 명시적 기술
□ 기존 이론에 도전하는 발견 부각
□ Boundary condition 확장 논의
□ 새로운 메커니즘/과정 제시
```

### 패턴 5: "Sample Size/Selection Issues"
```
리뷰어 코멘트:
"With only N participants, how can the authors
claim theoretical saturation?"

원인 분석:
├── Information power 논의 부재
├── 표집 논리 미흡
├── 포화 기준 불명확
└── 맥락 다양성 제한

대응 전략:
□ Malterud's "Information Power" 인용
□ 이론적 표집 논리 상세화
□ 포화 판단 근거 제시
□ 풍부한 데이터 품질 강조
```

### 패턴 6: "Researcher Bias/Subjectivity"
```
리뷰어 코멘트:
"How do we know the findings aren't simply
reflecting the researcher's preconceptions?"

원인 분석:
├── Reflexivity 논의 부재
├── 연구자 positionality 미기술
├── 대안적 해석 미검토
└── 확증편향 통제 방법 없음

대응 전략:
□ Reflexivity 섹션 추가
□ Positionality statement 포함
□ 동료 디브리핑 기술
□ 반대 사례(negative cases) 탐색 보고
```

## 3. R&R 실패 패턴

### 패턴 7: "Didn't Address Core Concerns"
```
에디터 결정:
"Despite revisions, the fundamental issues raised
by reviewers remain unaddressed..."

원인 분석:
├── 표면적 수정에 그침
├── 핵심 이슈 회피
├── Response letter 불성실
└── 추가 데이터/분석 거부

대응 전략:
□ 모든 코멘트에 구체적 대응
□ "We agree..." 또는 "We respectfully disagree..."
□ 변경 사항 페이지/라인 명시
□ 대안적 접근 시도 기록
```

### 패턴 8: "Theory Too Thin"
```
리뷰어 코멘트:
"The theoretical development remains underdeveloped.
The concepts need more elaboration..."

원인 분석:
├── 개념 정의 불명확
├── 개념 간 관계 미설명
├── 메커니즘 불분명
└── 이론적 통합 부족

대응 전략:
□ 각 개념 상세 정의 추가
□ 개념 간 관계 도식화
□ 작동 메커니즘 설명
□ 기존 이론과 연결
```

## 4. 방법론 관련 주요 이슈

### 이슈 A: Grounded Theory 오용
```
흔한 실수:
- 문헌검토 과다 (vs theoretical sensitivity)
- 코딩 절차 불명확
- Theoretical sampling 미적용
- 포화 주장 근거 없음

수정 전략:
1. 문헌검토 범위 조정
2. Open-Axial-Selective 코딩 명시
3. 표집 결정 과정 기술
4. 코드/범주 수 보고 + 포화 근거
```

### 이슈 B: Case Study 혼란
```
흔한 실수:
- 단일 사례 정당화 실패
- Yin vs Stake 혼용
- 분석 단위 불명확
- 일반화 주장 과다

수정 전략:
1. 사례 선정 논리 강화 (extreme, critical, typical)
2. 특정 전통 선택 및 일관성 유지
3. 분석 단위 명확히 정의
4. Analytical generalization 프레이밍
```

### 이슈 C: Phenomenology 오해
```
흔한 실수:
- 철학적 기반 무시
- 단순 주제 분석과 혼동
- Bracketing 논의 누락
- 본질 기술 부재

수정 전략:
1. Husserl vs Heidegger 명시
2. 현상학적 환원 절차 기술
3. Epoché/bracketing 방법 설명
4. 경험의 본질적 구조 도출
```

## 5. 주제별 리젝션 분석

### 경영학 (AMJ, ASQ, JMS)
```
Top Rejection Reasons:
1. Contribution weak (40%)
2. Methodology concerns (25%)
3. Wrong fit (20%)
4. Writing quality (15%)

성공 공식:
= Compelling phenomenon
+ Clear gap
+ Rigorous method (Gioia often)
+ Multiple contributions
+ Beautiful writing
```

### 교육학 (AERJ, RER)
```
Top Rejection Reasons:
1. Theoretical framework weak (35%)
2. Policy relevance unclear (25%)
3. Sampling issues (20%)
4. Ethics concerns (20%)

성공 공식:
= Strong theoretical lens
+ Clear educational implications
+ Diverse participant voices
+ Explicit positionality
```

### 의료/간호학 (QHR, JAN)
```
Top Rejection Reasons:
1. COREQ compliance (30%)
2. Rigor unclear (25%)
3. Clinical relevance (25%)
4. Ethics/consent (20%)

성공 공식:
= COREQ 체크리스트 완전 준수
+ Practice implications 명확
+ Patient/participant voice 중심
+ Ethical considerations 상세
```

## 6. 리젝션 후 전략

### 즉시 대응 (24-48시간)
```
하지 말 것:
✗ 감정적 반응
✗ 급한 resubmit
✗ 에디터에게 항의

해야 할 것:
✓ 피드백 냉정하게 읽기
✓ 핵심 이슈 정리
✓ 개선 가능성 평가
```

### 분석 및 개선 (1-2주)
```
단계:
1. 피드백 유형화
   - 수용할 것
   - 논쟁할 것
   - 명확화할 것

2. 개선 계획 수립
   - 대폭 수정 필요 여부
   - 추가 데이터 필요 여부
   - 저널 변경 필요 여부

3. 동료 자문
   - 피드백 공유
   - 의견 수렴
   - 전략 논의
```

### 재제출 결정
```
같은 저널 Resubmit:
- 수정 가능한 이슈일 때
- 에디터 긍정적 톤일 때
- Reviewer 유지될 때

다른 저널 Submit:
- 근본적 fit 문제일 때
- 완전 재작성 필요할 때
- 더 적합한 저널 있을 때
```

## 7. 예방적 체크리스트

### 제출 전 자가 점검
```
Contribution:
□ 3개 이상 명확한 기여 기술?
□ "So what?" 질문에 답?
□ 기존 연구와 차별화 명시?

Methodology:
□ 방법론 선택 정당화?
□ 분석 절차 투명?
□ 신뢰성 확보 방법 기술?
□ 데이터 구조표/시각화 포함?

Theory:
□ 개념 명확히 정의?
□ 개념 간 관계 설명?
□ 기존 이론과 연결?
□ Boundary conditions 명시?

Writing:
□ 첫 3페이지가 compelling?
□ 논리적 흐름?
□ 중복/반복 제거?
□ Native check 완료?
```

## 8. 결론: 성공의 공식

```
성공적인 질적연구 논문 = 
  Compelling Phenomenon
  + Clear Theoretical Gap
  + Rigorous Methodology
  + Surprising Findings
  + Multiple Contributions
  + Beautiful Writing
  + Right Journal Fit
  + Persistence
```

**Remember**: 최고의 질적연구자도 리젝션을 받는다. 
차이는 피드백에서 배우고 계속 개선하는 것이다.
