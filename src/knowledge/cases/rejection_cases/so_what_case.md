# "So What?" 리젝션 사례 분석

## 패턴 개요

### 정의
"So What?" 리젝션은 논문의 이론적/실무적 기여가 명확하지 않거나 중요하지 않다고 판단될 때 발생합니다.

### 전형적인 리뷰어 코멘트
```
"The contribution of this paper is unclear."
"I fail to see why this matters."
"What does this add to existing knowledge?"
"This seems incremental at best."
"The theoretical contribution is not compelling."
```

## 실제 사례

### 사례 1: 개념 재정의 논문

#### 원본 제출
```
제목: "재정의하는 조직 정체성: 새로운 관점"

서론 (요약):
"조직 정체성은 중요한 개념이다.
기존 연구들은 조직 정체성을 다양하게 정의해왔다.
본 연구는 조직 정체성의 새로운 정의를 제안한다."
```

#### 리뷰어 피드백
```
Reviewer 1:
"저자가 제안하는 '새로운' 정의가 기존 Albert & Whetten(1985)의
정의와 어떻게 다른지 명확하지 않습니다. 왜 새로운 정의가
필요한지에 대한 설득력 있는 논거가 없습니다."

Reviewer 2:
"So what if we adopt this new definition? What new insights
or research directions does it enable? The paper reads like
a semantic exercise without substantive implications."

Action Editor:
"두 리뷰어 모두 논문의 기여에 대해 의문을 제기하고 있습니다.
재정의 자체가 기여가 되려면, 그것이 열어주는 새로운 연구
가능성이나 기존 문제 해결을 명확히 보여주어야 합니다."
```

#### 문제 진단
```
1. Gap-Contribution 연결 부재
   - 왜 기존 정의가 문제인지 불명확
   - 새 정의가 해결하는 문제 불명시

2. "So What" 질문에 대한 답변 부재
   - 새 정의의 이론적 함의 없음
   - 실무적 적용 가능성 없음
   - 열리는 연구 방향 없음

3. 기존 문헌과의 차별화 실패
   - 표면적 차이만 주장
   - 본질적 차이 논증 부족
```

#### 수정 전략
```
1. 서론 재구성
   Before: "조직 정체성은 중요하다"
   After: "조직 정체성 연구는 [특정 문제]를 설명하지 못하며,
          이는 기존 정의의 [특정 한계] 때문이다"

2. 기여 명시화
   "본 논문의 기여는 세 가지다:
   (1) 기존 정의의 [한계]를 극복하는 새 정의 제안
   (2) 이를 통해 [현상 X]의 새로운 설명 가능
   (3) [연구 방향 Y]로의 확장 기반 마련"

3. 함의 구체화
   - 이론적: 어떤 새로운 연구 질문이 가능해지는가?
   - 실무적: 관리자에게 어떤 통찰을 주는가?
```

### 사례 2: 프레임워크 통합 논문

#### 원본 제출
```
제목: "X, Y, Z 이론의 통합 프레임워크"

서론 (요약):
"X, Y, Z는 모두 중요한 이론이다.
그러나 이들은 분절적으로 연구되어 왔다.
본 연구는 이 세 이론을 통합하는 프레임워크를 제안한다."
```

#### 리뷰어 피드백
```
Reviewer 1:
"통합의 필요성이 납득되지 않습니다. 저자는 단순히 세 이론이
'분절적'이라고 주장하지만, 그것이 왜 문제인지 설명하지
않습니다. 각 이론이 자체적으로 잘 작동한다면, 통합이
필요한 이유는 무엇입니까?"

Reviewer 2:
"I see what the authors are trying to do, but I don't see
why it matters. The framework appears to simply place three
theories side by side without generating new insights. What
can we explain with this framework that we couldn't before?"
```

#### 문제 진단
```
1. "왜 통합인가?"에 대한 답변 부재
   - 분절성 자체가 문제라는 논증 부족
   - 통합으로 얻는 것이 불분명

2. 통합의 부가가치 불명확
   - 1+1+1 = 3 (단순 합)
   - 1+1+1 > 3 (시너지) 미입증

3. "New Insight" 부재
   - 통합 프레임워크로 설명 가능한 새로운 것 없음
```

#### 수정 전략
```
1. 문제 상황 구체화
   "기존에 X, Y, Z 이론은 각각 [현상 A]를 설명했다.
   그러나 [현상 B]는 어느 이론으로도 설명되지 않는다.
   이는 세 이론 각각이 [특정 측면]만 포착하기 때문이다."

2. 통합의 부가가치 명시
   "세 이론의 통합은 다음을 가능하게 한다:
   (1) [현상 B]에 대한 설명
   (2) [이론 간 모순]의 해소
   (3) [새로운 명제]의 도출"

3. 새로운 통찰 제시
   - 통합으로 인해 가능해지는 새로운 예측
   - 개별 이론으로는 불가능했던 설명
```

## 일반적인 대응 전략

### 서론 강화
```
1. Hook을 문제/퍼즐로 시작
   "왜 X가 발생하는데 Y는 실패하는가?"

2. Gap을 구체적으로 명시
   "기존 연구는 A를 설명하지 못한다"

3. Contribution을 명확히 진술
   "본 연구는 세 가지 기여를 한다: ..."
```

### Discussion 강화
```
1. 이론적 함의 구체화
   - 어떤 이론에 어떻게 기여하는가?
   - 어떤 논쟁을 해결하는가?
   - 어떤 새로운 질문을 제기하는가?

2. 실무적 함의 구체화
   - 관리자가 무엇을 다르게 해야 하는가?
   - 조직에 어떤 변화를 가져오는가?
```

### Response Letter 작성
```
"We thank the reviewer for pushing us to articulate our
contribution more clearly. In the revised manuscript, we have:

1. Added a clear statement of contribution in the introduction
   (page 3, lines 15-25)

2. Specified what our framework enables that was not possible
   before (page 15, new section 'Theoretical Implications')

3. Included a table comparing our framework with existing
   approaches and highlighting unique insights (Table 2)"
```

## 체크리스트: "So What?" 방지

### 제출 전 자가점검
- [ ] 1문장으로 기여를 설명할 수 있는가?
- [ ] "왜 중요한가?" 질문에 구체적으로 답할 수 있는가?
- [ ] 기존 연구와의 차별점이 명확한가?
- [ ] 새로운 통찰/예측/설명이 있는가?
- [ ] 이론적 함의가 구체적인가?
- [ ] 실무적 함의가 actionable한가?

### 동료에게 물어볼 질문
- "이 논문이 왜 중요한지 알겠어?"
- "기존 연구와 뭐가 다른지 설명해 줄래?"
- "이 논문이 출판되면 뭐가 달라져?"
