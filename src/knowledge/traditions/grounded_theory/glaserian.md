# 고전적 근거이론 (Classic/Glaserian Grounded Theory)

## Barney Glaser의 접근

## 역사적 배경

### 기원
- 1967년 Glaser & Strauss: "The Discovery of Grounded Theory"
- 실증주의에 대한 반동
- 이론의 "발견" 강조

### Glaser의 배경
- Columbia University 양적 연구 훈련
- Paul Lazarsfeld의 영향
- 체계적 분석 강조

## 핵심 원칙

### 이론적 민감성 (Theoretical Sensitivity)
- 데이터에서 의미를 포착하는 능력
- 문헌 리뷰의 최소화 (사전에)
- 개념적 통찰력

### All is Data
- 모든 것이 데이터가 될 수 있음
- 면담, 관찰, 문서, 비공식 대화
- 연구자의 경험도 포함

### 출현 (Emergence)
- 이론은 데이터에서 "출현"
- 강제하지 않음 (No forcing)
- 자연스러운 발견

## 코딩 절차

### 1단계: 개방 코딩 (Open Coding)
- 줄 단위 코딩 (Line-by-line)
- 데이터를 조각내기
- 개념 생성

**코딩 질문**:
- "이 데이터가 무엇에 대한 연구인가?"
- "이 데이터가 어떤 범주를 나타내는가?"
- "실제로 무슨 일이 일어나고 있는가?"

**예시**:
```
데이터: "처음엔 아무것도 모른 채 시작했어요"
코드: 무지 속 시작, 초보자 상태, 준비 없음
```

### 2단계: 선택 코딩 (Selective Coding)
- 핵심 범주 (Core Category) 선정
- 핵심 범주와 관련된 코딩에 집중
- 이론적 포화까지 진행

**핵심 범주의 특성**:
- 대부분의 데이터와 연결
- 반복적으로 출현
- 다른 범주들을 통합
- 설명력이 높음

### 3단계: 이론적 코딩 (Theoretical Coding)
- 범주들 간의 관계 개념화
- 코딩 패밀리 (Coding Families) 활용
- 실질 이론 (Substantive Theory) 구축

**Glaser의 코딩 패밀리 예**:
- The Six C's: Causes, Contexts, Contingencies, Consequences, Covariances, Conditions
- Process: Stages, Phases, Progressions
- Strategy: Strategies, Tactics, Techniques
- Type: Types, Classes, Kinds

## 이론적 표집 (Theoretical Sampling)

### 원칙
- 이론 개발을 위한 표집
- 분석 결과가 다음 표집 결정
- 이론적 포화까지 진행

### 절차
1. 초기 표집: 관심 현상 경험자
2. 범주 발견 후: 범주 속성 탐색을 위한 표집
3. 포화 단계: 변이 확인을 위한 표집

## 지속적 비교 방법 (Constant Comparison)

### 비교의 수준
1. **사건 대 사건**: 초기 범주 형성
2. **사건 대 범주**: 범주 속성 발전
3. **범주 대 범주**: 이론적 통합

### 비교 과정
```
데이터 → 코드 → 개념 → 범주 → 이론
   ↑__________________________|
        지속적 비교와 수정
```

## 메모 작성 (Memoing)

### 메모의 유형
- **코드 메모**: 코드의 의미와 속성
- **이론적 메모**: 범주 간 관계, 가설
- **조작적 메모**: 연구 절차와 결정

### 메모의 특성
- 비공식적, 자유로운 형식
- 아이디어 기록
- 이론 개발의 기록

## 이론적 포화 (Theoretical Saturation)

### 정의
- 새로운 데이터가 새로운 이론적 통찰을 제공하지 않음
- 범주가 잘 발달하고 관계가 확립됨
- 이론이 데이터를 잘 설명함

### 판단 기준
- 범주의 속성이 충분히 발달
- 범주 간 관계가 명확
- 변이가 충분히 설명됨

## 문헌 리뷰의 위치

### Glaser의 입장
- **사전 리뷰 최소화**: 선입견 방지
- **이론 출현 후 리뷰**: 이론과의 연결
- **문헌을 데이터로**: 비교 대상

### 논쟁
- 현실적 어려움 (학위논문 등)
- 이론적 민감성 vs 오염
- 분야별 차이

## 결과물: 실질 이론

### 특성
- 특정 영역에 적용
- 데이터에 근거
- 수정 가능성 (Modifiable)

### 평가 기준 (Glaser의 4가지)
1. **적합성 (Fit)**: 데이터에 부합
2. **작동성 (Work)**: 현상을 설명
3. **관련성 (Relevance)**: 참여자에게 의미
4. **수정가능성 (Modifiability)**: 새 데이터에 열림

## 한계와 비판

### 방법론적 도전
- "발견"의 인식론적 가정
- 연구자 주관성 문제
- 절차의 모호함

### Strauss와의 분열
- Strauss & Corbin의 체계화에 대한 비판
- "강제"와 "출현"의 논쟁
- 패러다임 모델 거부
