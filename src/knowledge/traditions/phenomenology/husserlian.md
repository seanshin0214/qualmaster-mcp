# 기술적 현상학 (Descriptive/Transcendental Phenomenology)

## Edmund Husserl의 접근

## 핵심 개념

### 현상학적 환원 (Phenomenological Reduction)

#### 에포케 (Epoché)
- 자연적 태도의 중지
- 판단 보류 (Bracketing)
- 선입견과 가정의 괄호치기

#### 환원의 단계
1. **현상학적 환원**: 세계 존재에 대한 판단 중지
2. **본질 환원**: 개별에서 본질로
3. **초월론적 환원**: 순수 의식으로

### 지향성 (Intentionality)
- 의식은 항상 무엇에 '대한' 의식
- 노에시스 (Noesis): 의식 작용
- 노에마 (Noema): 의식 대상

### 본질 직관 (Eidetic Intuition)
- 자유 변경 (Free Variation)을 통한 본질 파악
- 불변의 구조 발견
- 현상의 본질적 특성 확인

## 연구 절차

### Colaizzi 방법 (1978)
1. 모든 기술 읽기
2. 의미 있는 진술 추출
3. 의미 공식화
4. 주제 묶음 조직
5. 철저한 기술 작성
6. 기본 구조 진술
7. 참여자 확인

### Giorgi 방법 (1985)
1. 전체 기술 읽기
2. 의미 단위 구분
3. 참여자 언어를 학문적 언어로 전환
4. 의미 단위를 구조로 통합
5. 구조에 대한 일반적 기술

### Moustakas 방법 (1994)
1. 에포케 수행
2. 현상학적 환원
3. 상상적 변형
4. 의미와 본질 통합

## 데이터 수집

### 심층 면담 특성
- 비구조화 또는 반구조화
- 열린 질문: "당신의 경험을 이야기해 주세요"
- 경험의 풍부한 기술 유도

### 면담 질문 예시
```
1. "이 경험을 처음 하셨을 때를 떠올려 주세요.
   어떤 일이 있었나요?"
2. "그 순간 어떤 생각이 드셨나요?"
3. "신체적으로 어떤 느낌이 있었나요?"
4. "그 경험이 당신에게 어떤 의미인가요?"
```

### 참여자 선정
- 경험 보유자 (Lived Experience)
- 경험을 표현할 수 있는 능력
- 연구 참여 의지
- 일반적으로 5-25명

## 분석 기법

### 수평화 (Horizontalization)
- 모든 진술을 동등하게 취급
- 중복 제거
- 의미 있는 진술 확인

### 텍스처 기술 (Textural Description)
- "무엇을" 경험했는가
- 경험의 내용
- 의식에 나타난 것

### 구조 기술 (Structural Description)
- "어떻게" 경험했는가
- 경험의 맥락과 조건
- 의미의 구조

### 본질 기술 (Essence Description)
- 텍스처와 구조의 통합
- 경험의 본질적 구조
- 불변의 의미 구조

## 엄격성 확보

### Bracketing 전략
- 연구자 가정 명시
- 메모와 저널링
- 동료 검토
- 지속적 반성

### 타당도 확보
- 참여자 확인 (Member Checking)
- 풍부한 기술 (Thick Description)
- 감사 추적 (Audit Trail)

## 적용 영역

### 적합한 연구 질문
- "~의 경험은 어떠한가?"
- "~의 본질은 무엇인가?"
- "~을 경험한다는 것의 의미는?"

### 적합한 분야
- 간호학: 환자 경험
- 교육학: 학습 경험
- 심리학: 정서적 경험
- 경영학: 조직 경험
