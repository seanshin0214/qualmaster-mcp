# 개념 공학 (Concept Engineering)

## 개념이란?

### 정의
**개념(Concept)**은 현상의 본질적 특성을 추상화한 것으로, 이론 구축의 기본 단위입니다.

### 좋은 개념의 특성
1. **추상성**: 특정 사례를 넘어 일반화
2. **명확성**: 정의가 분명
3. **구별성**: 유사 개념과 구별
4. **유용성**: 현상 설명에 기여

## 개념 개발 과정

### 1단계: 현상 관찰

**방법**
- 경험/관찰에서 패턴 발견
- 기존 이론의 한계 인식
- "이상한" 현상에 주목

**예시 질문**
- "기존 개념으로 설명되지 않는 것은?"
- "반복적으로 관찰되는 패턴은?"
- "왜 이런 현상이 발생하는가?"

### 2단계: 개념 정의

**정의 구성요소**
1. **속 (Genus)**: 상위 범주
2. **종차 (Differentia)**: 구별 특성

**예시: Psychological Safety**
```
속: 팀 내 공유된 믿음
종차: 대인관계 위험 감수가 안전하다는
정의: "팀 내에서 대인관계 위험을 감수하는 것이
       안전하다는 공유된 믿음" (Edmondson, 1999)
```

### 3단계: 개념 차원화

**차원 (Dimensions)**
- 개념의 하위 구성요소
- 다차원적 분해

**예시: Absorptive Capacity**
- 획득 (Acquisition)
- 동화 (Assimilation)
- 변환 (Transformation)
- 활용 (Exploitation)

### 4단계: 경계 설정

**무엇인가 vs 무엇이 아닌가**

| 개념 X는... | ≠ 유사 개념 Y |
|------------|--------------|
| 수준 명시 | 다른 분석 수준 |
| 초점 구별 | 다른 초점 |
| 메커니즘 구별 | 다른 작동 방식 |

**예시: Psychological Safety ≠ Trust**
- PS: 팀 수준 / Trust: 개인 간
- PS: 집단 규범 / Trust: 이자 관계
- PS: 발언 안전 / Trust: 상대 의존

### 5단계: 조작화

**측정 가능한 지표 개발**
- 설문 문항
- 관찰 지표
- 실험 조작

## 개념 명명 전략

### 좋은 이름의 특성
- **직관적**: 의미 추측 가능
- **기억하기 쉬움**: 간결
- **학술적**: 일상어 피함

### 명명 방법
1. **메타포 활용**: Absorptive Capacity (흡수 능력)
2. **합성어**: Psychological Safety
3. **기존 용어 전용**: Sensemaking
4. **신조어**: Ambidexterity

### 피해야 할 것
- 너무 긴 이름
- 이미 사용 중인 용어
- 부정적/편향된 함의

## 개념적 기여 유형

### MacInnis (2011) 분류

1. **식별 (Delineating)**: 새 개념 정의
2. **구별 (Differentiating)**: 유사 개념 구별
3. **통합 (Integrating)**: 기존 개념 결합
4. **일반화 (Generalizing)**: 적용 범위 확장
5. **재명명 (Revising)**: 기존 개념 수정

## 개념 논문 작성 팁

### 논의에서

- 실무적 함의
- 향후 연구 방향
