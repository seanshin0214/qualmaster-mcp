## 도구 활용 시나리오

### 시나리오 1: 연구 설계

```
User: "간호사의 소진 경험을 연구하려고 합니다"

→ suggest_methodology 호출:
  - research_question: "간호사의 소진 경험은 어떠한가?"
  - purpose: "explore"
  - participants: "중환자실 간호사"

→ 결과: 현상학 (IPA) 추천 + 근거 제시
```

### 시나리오 2: 데이터 분석

```
User: "이 면담 텍스트를 코딩해 주세요"
[면담 텍스트 첨부]

→ auto_coding 호출:
  - text: [면담 텍스트]
  - methodology: "phenomenology"

→ 결과: 코드 목록 + 근거
```

```
User: "지금까지 나온 코드들로 주제를 추출해 주세요"

→ extract_themes 호출:
  - codes: [코드 목록]
  - mode: "inductive"

→ 결과: 주제 구조 + 주제 정의
```

### 시나리오 3: 품질 평가

```
User: "우리 연구의 신뢰성을 평가해 주세요"

→ assess_quality 호출:
  - research_description: [연구 설명]
  - strategies_used: ["member_checking", "peer_debriefing"]
  - criteria: "lincoln_guba"

→ 결과: 4대 기준별 점수 + 개선 제안
```

### 시나리오 4: 논문 작성

```
User: "이 Introduction 섹션을 리뷰해 주세요"
[서론 텍스트]

→ review_paper 호출:
  - paper_text: [서론]
  - section: "introduction"

→ 결과: 100점 체크리스트 평가 + 구체적 피드백
```

### 시나리오 5: R&R 대응

```
User: "이 리뷰어 코멘트를 분석해 주세요"
[리뷰어 코멘트]

→ diagnose_rejection 호출:
  - reviewer_comments: [코멘트]

→ 결과: 패턴 진단 (예: "so_what")
```

```
User: "어떻게 대응해야 하나요?"

→ guide_revision 호출:
  - rejection_pattern: "so_what"
  - reviewer_comment: [코멘트]

→ 결과: 대응 전략 + 수정 제안
```

## AI 활용 윤리

### 원칙

1. **투명성**
   - AI 활용 사실 인지
   - 필요시 방법 섹션에 명시

2. **연구자 주도**
   - AI는 도구, 저자는 연구자
   - 최종 판단은 연구자

3. **비판적 수용**
   - AI 제안을 무조건 수용 금지
   - 학문적 판단 기준 적용

4. **표절 주의**
   - AI 생성 텍스트 그대로 사용 주의
   - 자신의 언어로 재작성

### 적절한 AI 활용

| 적절 | 부적절 |
|------|-------|
| 아이디어 브레인스토밍 | AI 생성 텍스트 그대로 제출 |
| 문헌 요약 보조 | 데이터 조작/생성 |
| 코딩 제안 참고 | AI 분석을 자신의 것처럼 |
| 글쓰기 피드백 | 연구자 해석 없이 결과 보고 |
| 편집/교정 | 표절 |

### 저널/학교 정책 확인

- 일부 저널은 AI 사용 공개 요구
- 학교별 학위논문 AI 정책 상이
- 투고 전 Author Guidelines 확인

## 효과적인 프롬프팅

### 기본 원칙

1. **맥락 제공**
   ```
   나는 [분야]의 [경험] 연구자입니다.
   [방법론]을 사용해 [주제]를 연구 중입니다.
   ```

2. **구체적 질문**
   ```
   (X) "코딩 도와주세요"
   (O) "이 면담 텍스트를 IPA 관점에서 경험적 진술을
        식별하고 초기 코드를 제안해 주세요"
   ```

3. **단계적 요청**
   ```
   1단계: 텍스트에서 의미 단위 식별
   2단계: 각 단위에 서술적 코드 부여
   3단계: 코드들의 패턴 탐색
   ```

4. **출력 형식 지정**
   ```
   다음 형식으로 제시해 주세요:
   | 원문 | 의미단위 | 코드 | 근거 |
   ```

### 프롬프트 예시

#### 방법론 질문
```
현상학과 근거이론의 차이점을 설명해 주세요.
특히 다음 측면에서:
1. 적합한 연구 질문
2. 데이터 분석 절차
3. 결과물 유형
```

#### 분석 요청
```
다음 면담 텍스트를 Giorgi의 기술적 현상학 방법으로
분석해 주세요:
1. 의미 단위 구분
2. 심리학적 언어로 전환
3. 구조 통합 제안

[면담 텍스트]
```

#### 논문 피드백
```
다음 서론을 AMR 수준의 기준으로 평가해 주세요:
1. Hook의 효과성
2. Gap의 설득력
3. 기여 진술의 명확성

점수와 구체적 개선 제안을 부탁합니다.

[서론 텍스트]
```

## 한계 인식

### AI가 잘하는 것
- 정보 검색/요약
- 패턴 인식
- 형식적 평가
- 대안 제시

### AI가 못하는 것
- 원천적 통찰
- 맥락적 판단
- 윤리적 결정
- 창의적 이론화

### 연구자의 역할
- 연구 질문 설정
- 참여자 관계
- 해석적 판단
- 학문적 책임
